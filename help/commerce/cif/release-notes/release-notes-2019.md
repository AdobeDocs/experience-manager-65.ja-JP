# Commerce統合フレームワークGitHubリリースの概要

## リリース日：2019年11月

| GitHub | バージョン | 詳細なリリースノート |
|:-------|:-----:|---------------------:|
| CIFコネクタ | 0.7.1 | [リリースノート](https://github.com/adobe/commerce-cif-connector/releases) |
| CIFコアコンポーネント | 0.6.0 | [リリースノート](https://github.com/adobe/aem-core-cif-components/releases) |
| CIFアーキタイプ | 0.6.2 | [リリースノート](https://github.com/adobe/aem-cif-project-archetype/releases) |

### 新機能 {#what-is-new-november}

* 作成者は、サイトエディターの新しい「製品/カテゴリで表示」オプションを使用して、製品の詳細ページと製品/カテゴリを含む製品リストページをプレビューできます。

* 作成者は、製品SKU別にアセットにタグを付けたり、SKU別に製品固有のアセットを検索したりできます。

* 買い物かごに追加されたクーポンサポートの追加/削除。

* Braintreeの支払いサポートがAEM Veniaストアフロントに追加されました。

### 改善点{#what-is-improved-november}

* カテゴリ/製品ピッカーが、マルチストア設定での指定されたMagentoストアビューに従って機能が強化されました。

* npmパッケージとして使用できるReactベースのコンポーネント。 これにより、開発者は、新しいReactプロジェクトの依存関係としてReactコンポーネントパッケージを使用して、既存のコンポーネントをカスタマイズしたり、新しいReactベースのコンポーネントを開発したりできます。

* GraphQLクエリのカスタマイズがシンプルになりました。 これにより、開発者は、より少ないコードでCIFコアコンポーネントをカスタマイズできます。

| GitHub | バージョン | 詳細なリリースノート |

### 新機能 {#what-is-new-october}

* 製品の詳細ページおよび製品リストページ用の完全にオーサリング可能なテンプレート。 作成者は、新しいテンプレートを作成し、これらのテンプレートに製品リストおよび製品の詳細コンポーネントをドラッグ&amp;ドロップできるようになりました。 作成者は、他のコンポーネントを追加するだけでなく、これらのテンプレートのレイアウトも変更できるようになり、マーケティングとコマースのコンテンツを組み合わせて驚くべきエクスペリエンスを無制限に作成できます。

* すべての作成者に適したCIFコアコンポーネントが、[AEMスタイルシステム](https://helpx.adobe.com/jp/experience-manager/6-5/sites/authoring/using/style-system.html)をサポートするように拡張されました。 製品リストコンポーネントのスタイルの例が提供されています。

* アカウント管理用のReactベースのクライアントサイドコンポーネント。 このリリースでは、次の機能がサポートされています。ログイン、パスワードを忘れた場合、およびアカウントを作成します。

### 改善点{#what-is-improved-october}

* 製品の詳細および製品リストのコンポーネントが強化され、作成者がテンプレートやページに配置されたときにレイアウトのプレビューを表示できるようになりました。

* MinicartおよびCheckoutコンポーネントで、拡張性を向上させるためにReactフックが使用されるようになりました。

### 新機能 {#what-is-new-september}

* 作成者が特定の製品の詳細ページまたは製品リストページをエンリッチメントできるマルチテンプレート機能。 作成者は、カスタムの製品詳細ページや製品リストページを簡単に作成し、製品やカテゴリピッカーを使用して、カスタムページを特定の製品やカテゴリに割り当てることができます。

* マルチカタログ連結を使用すると、作成者はAEM製品コンソールで複数のカタログを連結できます。 作成者は、連結を作成した後に、カタログ連結のプロパティを編集および表示することもできます。

* GraphQLを使用したReactベースのクライアント側チェックアウトとミニカートで、完全な基本的なショッピングジャーニーをサポート。

* チェックアウトコンポーネントには、住所フォーム、支払いの選択、配送方法の選択が含まれます。

### 改善点{#what-is-improved-september}

* 製品ティーザーおよび製品カルーセルコンポーネントは、製品のバリアントをサポートしています。

### 新機能 {#what-is-new-august}

* CIFアーキタイプへのCIFコネクタの埋め込みは、開発者がより柔軟に作業できるようにオプションになりました。

* CIFコンポーネントは、「Venia」固有のCSSスタイル設定から切り離されているので、開発者は選択したCSSスタイル設定を適用できます。

* マルチストア/サイト機能：複数のAEMサイト構造でCIFコアコンポーネントを使用でき、基盤となるGraphQLクライアント実装が異なるMagentoストア/ストア表示に接続できるようにします。

* HTTPGETを介した特定のGraphQLクエリに対してGraphQLキャッシュが有効になり、応答時間を短縮できます。

* 製品説明表示がAEM製品コンソールで有効になっている。

* コマースティーザーは、WCMティーザーコンポーネントを拡張し、作成者がCTAフィールドを製品の詳細ページまたは製品リストページに追加することもできるようにしました。

* 作成者がAEMページに配置して、AEMページ、製品の詳細ページ、製品リストページまたは外部リンクにリンクできるボタン。

### 改善点{#what-is-improved-august}

* Magentoストアの設定をOSGiからAEM製品コンソールに移動し、統合の設定を作成者に適したものにしました。

### 新機能 {#what-is-new-july}

* 最初のCIFアーキタイプで、開発者にいくつかのデプロイメントオプションを提供します。1. AEM Veniaストアフロントをデプロイします2. 新しいプロジェクトの基礎モードをデプロイする3. 既存のプロジェクトでのCIF要素の使用

* カテゴリやサブカテゴリ間のナビゲーションをサポートするマルチレベルのカタログナビゲーション。

* カテゴリページでのページネーション（UXの改善に役立つ）

* 動的属性のレンダリングをサポートするために、製品の詳細および製品リストコンポーネントでの価格属性のクライアント側レンダリング。

* サーバー側の製品カルーセル：カルーセル形式で紹介する製品のリストを表示します。

* サーバーサイドのおすすめカテゴリリスト：AEMページにカテゴリのリストを表示します。

* Magento2.3.2のサポートと、製品のプロパティに関するバグ修正が製品コンソールに表示されます。
