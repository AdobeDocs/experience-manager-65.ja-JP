# To Do リストの操作{#working-with-to-do-lists}

TODO リストを表示すると、自分に割り当てられているビジネスプロセスのタスク、自分が属しているグループ、または他のユーザーの共有タスクが表示される場合があります。 リクエストの承認または却下、詳細情報の追加など、必要に応じてタスクを開き、操作し、完了できます。 タスクを完了すると、ビジネスプロセスの次のユーザーにタスクが送信されます。

## TODO リストについて {#about-todo-lists}

AEM Forms Workspace には次の 3 種類の TODO リストがあります。

* 個々のリスト。ユーザーに直接割り当てられたタスクが含まれます。
* グループリスト：グループに割り当てられたタスクを含みます。 グループの任意のメンバーがタスクを開いて完了できます。 タスクを開くには、グループのメンバーが最初にタスクを要求する必要があります。
* 共有リスト。自分や他のユーザーと TODO リストを共有しているユーザーに割り当てられたタスクが含まれます。 リストを共有しているユーザーは、共有タスクを要求、開き、完了できます。

タスクの上にマウスポインターを置いたときに表示されるアイコンをクリックすると、タスクを開かずに一部のアクションを実行できます。

>[!NOTE]
>
>感嘆符のアイコンは、タスクの優先度が高いことを示します。

## 一般的なタスク {#typical-tasks}

タスクを開いて操作する場合、使用できるツールはタスクによって異なります。 異なるタスクを実行する場合は、異なるアクションを実行する必要があります。そのため、一部のツールを使用できる場合と使用できない場合があります。 受け取る一般的なタスクを以下に示します。

* **情報の提供**：フォームを完了して送信する必要があるタスクを受け取ります。

* **情報の確認**：情報を確認し、コンテンツに対するログオフを求めるタスクを受け取ります。

* **マルチユーザーレビュー**：他のユーザーがタスクを受け取るのと同時に、タスクを受け取ります。 自分と他のユーザーは、情報を提供するか、コンテンツをレビューするか、またはその両方をおこなう必要があります。 このタイプのタスクでは、次のツールを使用できます。

   * タスクの手順の表示
   * タスクを割り当てられているすべてのユーザーの完了ステータスの表示
   * タスクを割り当てられたすべてのユーザーのコメントの表示
   * 自分でタスクにコメントを追加する

上記のタスクのいずれかで使用できる追加のツールは、次のとおりです。

* 送る
* 共有
* 相談
* 戻る
* 備考
* 添付ファイル

## タスクを開く {#opening-tasks}

TODO リストのタスクを開いてロックしたり、グループまたは共有の TODO リストのタスクを要求して開くことができます。 タスクを開くと、メインウィンドウに表示されます。 その他のタスクは、TODO リストの横のタスクリストに表示されます。

タスクのサマリー URL が存在する場合は、タスクに関連付けられたフォームの代わりに、タスクのサマリービューがデフォルトで開きます。タスクの割り当てで「最大化モードでフォームを開く」オプションを有効にしても、フォームは最大化モードで開きません。

>[!NOTE]
>
>タスクを開くと、タスクのデフォルトに応じて、関連するフォームがフルビューで表示される場合があります。

### リストからタスクを開いてロックする {#open-and-lock-a-task-from-your-list}

TODO リストからタスクを開いたときに、リストが共有されている場合は、タスクをロックして、リストにアクセスできる別のユーザーがタスクを操作しないようにすることができます。

1. TODO ページの左側のウィンドウで、個々の TODO リストを選択します。 すべてのタスクが中央のウィンドウに表示されます。

   >[!NOTE]
   >
   >TODO リスト内でプロセスタイプを選択して、タスクをフィルタリングできます。 TODO リストを選択して、TODO リストのすべてのタスクを再度表示できます。

1. 必要に応じて、タスクをロックします。 タスクをロックするには、タスクの「すべてのオプション」アイコンをクリックし、「ロック」を選択します。 オプションを使用できるように、タスクにポインターを置きます。

   >[!NOTE]
   >
   >タスクが開いているときに、任意のタブでタスクをロックまたはロック解除することもできます。

   ![lock_task](assets/lock_task.png)

   タスクの [ すべてのオプション ] メニュー

1. タスクをクリックして開きます。

### 共有リストまたはグループリストからタスクを開いて要求する {#open-and-claim-a-task-from-a-shared-or-group-list}

グループまたは共有リストを開いてタスクを要求すると、そのタスクはグループまたは共有リストから個々の TODO リストに移動します。 リストへのアクセス権を持つ他のユーザーがタスクを操作するのを防ぎます。

1. TODO ページの左ペインで、グループまたは共有の TODO リストを選択します。すべてのタスクが中央のウィンドウに表示されます。
1. 次のいずれかの手順を実行します。

   * タスクをグループまたは共有の TODO リストから開かずに要求するには、タスクにポインターを重ねて「**要求**」をクリックします。あるいは、タスクが開いたら、タスクペインの下のアクションバーで「要求」ボタンを使用します。要求すると、タスクがグループまたは共有の TODO リストからリストに移動します。
   * グループまたは共有の TODO リストからタスクを要求して開くには、 **要求して開く**.

## タスクの使用 {#working-with-tasks}

タスクを開いた後、メインウィンドウに表示されるタブと、使用可能なツールは、タスクによって異なります。 表示されるタブを以下に示します。

* **タスクの概要**：タスクが開くと、タスクの割り当て手順のプロセスで指定された URL を使用して、タスクが存在する場合に、タスクに関する情報をタスクの概要ペインに表示できます。 タスクの「Task Summary Pane」追加および関連情報を使用して、AEM Forms Workspace のエンドユーザーに対してさらに多くの値を追加できます。 タスクの概要 URL が存在しない場合、このタブは使用できません。

* **詳細**：現在のタスクおよびタスクが属するプロセスに関する情報を提供します。

* **フォーム**：タスクに関連付けられているフォームを表示します。 フォームには、PDF、HTML、ガイド、SWFファイルなど、様々なファイルタイプが含まれます。 フォームは、通常の印刷可能なフォームや Web ベースのフォームのように見えたり、一連のウィザード形式のパネルを通じてユーザーに情報を収集させたりできます。

* **履歴**：プロセスインスタンスに含まれるタスク、および各タスクに関連付けられているフォーム、タスクの割り当て、添付ファイルがリストされます。

* **添付ファイル**：タスクに関連付けられている既存の添付ファイルを表示し、必要に応じて添付ファイルを追加します。

* **メモ**：タスクに関連付けられている既存のメモを表示し、必要に応じてメモを追加します。

タスクを操作する際に表示される可能性のあるツールと実行できるアクションを、以下に示します。

### タスクの転送、共有、問い合わせ {#forward-share-or-consult-on-a-task}

メモや添付ファイルと共にタスクを別のユーザーに転送したり、タスクを共有したり、別のユーザーにタスクを問い合わせたりできます。 タスクに関連付けられているフォームデータを変更する場合は、タスクを転送、共有、または問い合わせる前にフォームをドラフトとして保存します。 それ以外の場合は、更新されたフォームなしでタスクが送信されます。 タスクを転送して共有した後、タスクを受け取ったユーザーは、タスクを要求して完了したり、自分に返したりできます。 タスクを問い合わせると、ユーザーはタスクを返すことしかできません。

1. 保持するタスクに関連付けられたフォームを変更する場合は、 **保存**. 「保存」オプションは、各タブの下部にあるアクションバーで使用できます。 それ以外の場合は、更新されたフォームなしでタスクが送信されます。

   >[!NOTE]
   >
   >作業中のタスクによっては、一部のフォームで「保存」ボタンを使用できない場合があります。

1. 任意のタブで、次のいずれかのボタンをクリックします。

   * **送る**
   * **共有**
   * **相談**

   >[!NOTE]
   >
   >タスクによっては、タスクを開かずに TODO リストからこれらの操作を実行できる場合もあります。

1. ポップアップダイアログウィンドウで、タスクの転送、共有、問い合わせを行うユーザーの名前を検索して選択します。

### タスクを返す {#return-a-task}

1. 任意のタブで、 **戻る**. タスクは、以前にタスクを転送したユーザーの TODO リストに返されます。また、自分とタスクを共有したり、問い合わせたりしたユーザーの TODO リストにも返されます。

### タスクをオフラインにする {#take-a-task-offline}

タスクをオフラインで操作し、後でAdobe®Reader®、Adobe® Acrobat® Professional またはAdobe® Acrobat® Standard からフォームを送信することができます。 フォームが送信されると、電子メールクライアントは、適切なサーバーの電子メールアドレスで起動されます。 その後、入力したフォームをサーバーに電子メールで送信できます。

1. 任意のタブで、「**オフライン**」をクリックします。
1. フォームを保存するファイル名を指定し、「 **保存**. タスクに関連付けられているフォームはローカルに保存され、フォームが送信されるまで、タスクは TODO リストに残ります。

## タスクの完了 {#completing-tasks}

タスクを完了する方法は、タスク自体とプロセス内での役割によって異なります。 リクエストの承認または拒否、コンテンツの提供、情報の確認および検証を行う、またはアクションが実行されたことを示すように求められる場合があります。

タスクは、様々な方法で完了できます。

* 任意のタブで使用可能なアクションの使用
* フォーム自体に組み込まれたアクションの使用
* タスクを開かずに TODO リストから

>[!NOTE]
>
>このオプションは、プロセスの設計時に `Assign Task` 手順で `isMustOpenToComplete` フィールドを選択していない場合に使用できます。

* 電子メールによる通知（電子メールによる通知）

タスクを完了すると、タスクに応じて、アクションを確認する確認ダイアログボックスが表示される場合があります。 例えば、指定した情報の有効性を確認するダイアログが表示される場合があります。

>[!NOTE]
>
>タスクを変更したが完了する準備ができていない場合は、「保存」をクリックして作業内容をドラフトとして保存し、後で元の状態に戻すことができます。

### タスクを完了 {#complete-a-task}

1. 次のいずれかの手順を実行します。

   * タスクを選択し、リストの下部のプロセスで必要な次の手順の適切なボタンをクリックします。
   * フォームにボタンがなく、AEM Forms Workspace の「完了」ボタンを使用できる場合は、「**完了**」をクリックします。
   * フォームにボタンがありながら、AEM Forms Workspace の「完了」ボタンを使用できない場合は、プロセス内で次の手順に進むためのボタンをフォーム上でクリックします。

   フォームにボタンがなく、AEM Forms Workspace の「完了」ボタンを使用できない場合は、フォームを送信できないことを示すメッセージが表示されます。

1. 確認ダイアログボックスが表示されたら、次のいずれかの操作を行います。

   * クリック **OK** タスクを完了し、サインオフする準備が整っている場合。
   * クリック **キャンセル** タスクに戻り、サインオフする準備がまだ整っていない場合。

>[!NOTE]
>
>フォーム内でプロセスプロパティを使用すると、HTMLフォーム内に「送信」ボタンが表示される場合があります。 このボタンは、同じフォームがPDFとしてレンダリングされる場合は表示されません。 タスクを完了するには、AEM Formsワークスペースの下部にある、フォームの外側にある「送信」ボタンをクリックします。フォーム内の「送信」ボタンではクリックしません。

### タスクの一括承認 {#bulk-approve-tasks}

TODO リストから複数のタスクを送信できます。 同じプロセスのタスク、同じタスク名、同じルートオプションのみを一緒に送信できます。

>[!NOTE]
>
>このオプションは、プロセスを設計する際に、Workbench のタスクの割り当て手順で isMustOpenToComplete フィールドを選択していない場合に使用可能です。

1. TODO ページの左側のウィンドウで、個々の TODO リストを選択します。 すべてのタスクが中央のウィンドウに表示されます。
1. 選択 **一括モードを有効にする**. リスト内のタスクの前にチェックボックスが表示されます。

   >[!NOTE]
   >
   >このオプションは、プロセスを設計する際に、Workbench のタスクの割り当て手順で isMustOpenToComplete フィールドが選択されているタスクには使用できません。このようなタスクのチェックボックスは、TODO リスト内では常に無効になります。

1. 一括承認のタスクを選択します。 同じプロセスの複数のタスクを、同じタスク名と同じルートオプションで選択できます。 承認するタスクを 1 つ選択すると、同じプロセス、同じタスク名、同じルートオプションのタスクのみが有効になります。残りのタスクは無効になります。

   ![1_bulkapproval](assets/1_bulkapproval.png)

1. 使用可能な「送信」オプションをクリックします。 選択したタスクが送信されます。

   ![bulkapproval](assets/bulkapproval.png)

## 電子メールを使用したタスクへの参加 {#participating-in-tasks-through-email}

タスクは電子メールで受信し、完了することができます。 電子メールを使用してタスクに参加すると、TODO リストで新しいタスクを定期的に確認したり、追跡ページでタスクのステータスを確認したりする必要がなくなります。

まず、メール通知を受信するように AEM Forms Workspace の環境設定を行います。AEM Forms Workspace では、ユーザーの TODO リストまたはユーザーが所属するグループの TODO リストにあるタスクに関して、メール通知を送信できます。管理者は、電子メール通知メッセージを送信するタイミングと受信するユーザーを決定します。

電子メールメッセージには、AEM Forms Workspace でタスクを開くリンク、タスクに使用されるフォームの添付ファイル、電子メールを使用してタスクを完了するためのアクションが含まれる場合があります。 電子メールメッセージにフォームが含まれている場合は、フォームを開き、タスクを完了するためのボタンがフォームに組み込まれている場合は、タスクを完了することができます。 タスクを完了するためのアクションが電子メールメッセージに含まれている場合、電子メール内のアクションをクリックするか、電子メール本文の最初の行に入力したアクションを使用して電子メールに返信することで、タスクを完了できます。

>[!NOTE]
>
>* 適切なメールのテンプレートを使用するように Workspace を設定するには、[AEM Forms JEE 管理者ガイド](https://help.adobe.com/en_US/AEMForms/6.1/AdminHelp/)を参照してください。
>
>* AEM Forms Workspace にタスクを送信した後にドラフトが転送されると、メール通知が送信されます。 ドラフトが AEM Forms Workspace のスタートポイントから転送される場合、メール通知は送信されません。

メールを使用してタスクを完了すると、そのタスクは AEM Forms Workspace の TODO リストから削除されます。

>[!NOTE]
>
>ブラウザーで AEM Forms Workspace にログインせずに TODO タスクへのリンクを開いた場合は、TODO リンクに直接アクセスすることに失敗して例外が表示されます。メールのリンクをクリックする前に、AEM Forms Workspace にログインしてください。

>[!NOTE]
>
>電子メール通知を転送してタスクを他のユーザーに割り当てることはできません。 タスクは、AEM Forms Workspace 内から他のユーザーにのみ転送できます。

### 電子メール通知メッセージを受信 {#receive-email-notification-messages}

1. クリック **環境設定**.
1. Adobe Analytics の **タスクイベントを電子メールで通知** リスト、選択 **はい**.
1. 電子メールメッセージにフォームとデータを含めるには、 **Formsをメールに添付** リスト、選択 **はい**.

## モバイルデバイスを通じたタスクへの参加 {#participating-in-tasks-through-mobile-devices}

AEM Forms Workspace アプリを使用して、モバイルデバイスからタスクに参加できます。 アプリケーションをインストールする前に、システム管理者に問い合わせて、組織がAEM Forms workspace アプリの使用をサポートしていることを確認してください。

## デッドラインとリマインダーについて {#about-deadlines-and-reminders}

「*デッドライン*」は、タスクの完了期限となる日時を決定します。デッドラインが過ぎると、サーバーはタスクをプロセス内の次のステップ（別のユーザーの TODO リスト）にルーティングし、デッドラインアイコンをタスクに表示します。 デッドラインアイコンは、プロセスに関連付けられているルールに関係なく表示されます。

「*リマインダー*」は、ユーザーが注意を向ける必要があるタスクをユーザーに通知します。リマインダーは、あらかじめ定義された時間に発生し、関連するタスクを完了するまで一定の間隔で発生します。 リマインダーを受け取ると、リマインダーアイコンがタスクに表示されます。

ビジネス・プロセスは、期限とリマインダの動作とタイミングを決定します。 すべてのプロセスにデッドラインとリマインダーがあるわけではありません。 管理者は、デッドラインとリマインダーに関する電子メール通知を送信するかどうかを指定します。 電子メール通知を受信するかどうかを設定できます。

## グループキューと共有キューのタスクの操作 {#working-with-tasks-from-group-and-shared-queues}

自分に割り当てられているすべてのタスクが TODO リスト（キュー）に表示されます。

アクセス権のあるすべてのグループおよび共有 TODO リストは、TODO ページの左側のウィンドウにも表示されます。 アクセス権のある任意の TODO リストからタスクを完了できます。

1 つのグループ TODO リストに複数のメンバーを含めることができます。 管理者が、組織の特定の要件に基づいてグループ TODO リストを設定します。 グループ TODO リストを使用すると、同じような責任を共有する複数のユーザーに作業を配布できます。

例えば、チームの各メンバーがローン申し込みフォームを処理するとします。 これらのタスクはすべて、グループのすべてのメンバーがアクセスできるグループ TODO リストに送信されます。 グループの各メンバーは、その TODO リストからタスクにアクセスできます。

共有 TODO リストは、別のユーザーから TODO リストを共有されたとき、または明示的にあるタスクを共有されたときに表示されます。その後、そのユーザーの TODO リストでタスクを表示し、そのユーザーに代わってタスクを完了することができます。 例えば、休暇を取る場合、TODO リストを同僚と共有して、不在時にタスクを完了するように選択できます。

>[!NOTE]
>
>不在設定を指定して、不在中に他のユーザーにタスクを転送することもできます。

グループまたは共有の TODO リストからタスクを操作するには、まずタスクを要求します。 その後、タスクを完了するか別のユーザーに転送するまで、タスクの所有者になります。

### キューの共有 {#sharing-queues}

TODO リストを別のユーザーと共有し、そのユーザーが TODO リストで新しいタスクを表示して操作できるようになります。 TODO リストを共有する前に TODO リストにタスクが存在する場合、他のユーザーはタスクを表示できません。 TODO リストへのアクセス権を付与した後、ユーザーは TODO リストに追加されたタスクのみを表示および要求できます。

ユーザーが共有キューにタスクを表示するには、プロセスデザイナーが User Service の「Task Access Control List(ACL)」タブの「Add ACL for Shared Queue」オプションを有効にする必要があります。

>[!NOTE]
>
>不在設定を指定して、TODO リスト全体を共有する代わりに、不在時に他のユーザーにタスクを転送することもできます。

**キューを共有**

1. Adobe Analytics の **キュー** 」タブをクリックします。 **環境設定** タブで、「+」アイコン（「現在自分のキューを共有しているユーザー」）をクリックします。
1. ユーザーの名前を検索して選択します。
1. クリック **共有** 」ボタンをクリックして、選択したユーザーとキューを共有します。
1. ユーザーの名前を選択し、 **共有**.

   >[!NOTE]
   >
   >TODO リストの共有からユーザーを削除するには、次をクリックします： **X** ユーザーが表示される行の最後にあるアイコン。

### 他のキューへのアクセス {#accessing-other-queues}

別のユーザーの TODO リストへのアクセス権を要求して、ユーザーの TODO リストの新しいタスクを表示して要求できます。

別のユーザーの TODO リストへのアクセス権を要求すると、そのユーザーの TODO リストに、要求を承認または拒否するタスクが表示されます。 ユーザーがタスクを完了すると、TODO リストに通知が届きます。

別のユーザーの TODO リストへのアクセス権を付与されている場合、アクセス権を付与される前にユーザーの TODO リストに存在していたタスクを表示することはできません。 TODO リストへのアクセス権を付与された後にユーザーの TODO リストに追加されたタスクのみを表示できます。

**別のキューへのアクセス**

1. 「**環境設定**」タブで、「**キュー**」タブを開きます。
1. 「アクセスできるユーザーキュー」で「+」をクリックします。ポップアップダイアログでユーザーの名前を検索します。
1. ユーザーの名前を選択し、 **リクエスト**.

   >[!NOTE]
   >
   >別の TODO リストへのアクセスを削除するには、アクセス権を持つユーザーキューからユーザー名を選択し、 **X** ユーザー名に言及している行の最後に配置されます。 TODO リストへのアクセス要求が保留中の場合は、別の TODO リストへのアクセスを削除できません。

## 不在時の環境設定の指定 {#setting-out-of-office-preferences}

不在を予定している場合は、その期間に割り当てられたタスクに対する処理を指定できます。

不在設定が有効になる開始日時および終了日時を指定するオプションがあります。サーバーとは異なるタイムゾーンにいる場合、サーバーのタイムゾーンが使用されます。

すべてのタスクを送信するデフォルトの担当者を設定できます。 また、特定のプロセスのタスクを別のユーザーに送信したり、戻るまで TODO リストに残したりする場合に、例外を指定することもできます。 指定された人物が不在の場合は、その人物が指定したユーザーにタスクが送信されます。 タスクを不在以外のユーザーに割り当てることができない場合、そのタスクは TODO リストに残ります。

>[!NOTE]
>
>不在時には、以前 TODO リストにあったタスクはそのまま残り、他のユーザーには転送されません。

### 不在時の環境設定を設定 {#set-out-of-office-preferences}

1. クリック **環境設定** をクリックします。 **不在**.
1. 不在時期を指定するには、次のいずれかの手順を実行します。

   * 不在を無期限に指定するには、「**現在の状態**」リストで「**不在**」を選択し、日付範囲を追加しないようにします。
   * 不在の開始日時を指定し、次の期間を「+」でクリックします： **不在スケジュール**. カレンダーと時間リストを使用して、開始日時を指定します。 終了日時を指定しない場合、環境設定を変更するまで、開始日時から無期限に不在と見なされます。

1. タスクをデフォルトで処理する方法を指定するには、次のいずれかのオプションを **不在時：不在タスクのデフォルトのユーザー** リスト：

   * 選択 **割り当てなし** タスクを TODO リストに残して戻る
   * 選択 **ユーザーを検索** タスクを割り当てるユーザーを検索します。 ユーザーを選択すると、不在スケジュールを表示することもできます。

1. 例外をデフォルトに設定するには、「+」をクリックして、 **例外を処理**、例外を作成するプロセスを選択し、別のユーザーを選択するか、「 **割り当てなし** から **が割り当てられています** リスト。

   >[!NOTE]
   >
   >プロセスデザイナーは、一部のプロセスのタスクを常に非公開にし、他のユーザーに転送しないように指定できます。 この設定は、行った設定よりも優先されます。

1. 環境設定の指定が完了したら、「**保存**」をクリックします。現在不在であることを設定した場合、変更は直ちに有効になります。 それ以外の場合は、指定した開始日時に有効になります。 不在中にログインしても、設定を変更するまでは不在と見なされます。
