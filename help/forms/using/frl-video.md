# ビデオスクリプト：AEM Forms 上での Adobe Acrobat の機能制限ライセンス（FRL）の設定

| ビジュアル | タイトル | 話者スクリプト |
|--------|-------|---------------|
| AEM Forms ロゴと FRL グラフィックを含むタイトル画面 | はじめに | AEM Forms サーバー上での Adobe Acrobat の機能制限ライセンス（FRL）の設定に関するこのチュートリアルへようこそ。アドビが永続ライセンス版から新しい FRL モデルに移行する際には、このプロセスが不可欠です。 |
| 矢印が FRL のトランジションを指す、EOL の日付を示すタイムライン | 背景 | Adobe Acrobat Classic バージョン永続ライセンス版は提供終了に達しつつあり、新しいバージョンへの移行が必要になります。AEM Forms PDF Generator は、ドキュメントコンバージョン機能に対して Adobe Acrobat に依存します。機能制限ライセンスモデルは、同じ機能を提供しますが、Adobe Admin Console を通じて管理される最新のライセンスアプローチを備えています。 |
| チェックボックスとアイコン付きの必須項目のチェックリスト | 前提条件 | 開始する前に、必要な項目がすべて揃っていることを確認しましょう。1 つ目は、システム管理者権限を含む Adobe Admin Console へのアクセス権です。2 つ目は、Admin Console でデプロイメント管理者の役割を持つユーザーアカウントです。3 つ目は、AEM Forms を実行しているサーバーに対するローカル管理者権限です。4 つ目は、AEM Forms サーバー上での Windows 64 ビット版オペレーティングシステムです。5 つ目は、ライセンスアクティベーション用の安定したインターネット接続です。また、Adobe Admin Console インターフェイスに精通し、AEM Forms のデプロイメントアーキテクチャを理解している必要があります。 |
| バックアップグラフィック付きの警告アイコン | 重要な注意事項 | このプロセス中に既存の Acrobat インストールをアンインストールする必要があるので、開始する前にカスタム Acrobat 設定を必ずバックアップしてください。 |
| 1～7 の番号付き手順を示すフロー図 | プロセスの概要 | このプロセスには、Admin Console での FRL パッケージの準備、ダウンロード権限の付与、以前の Acrobat バージョンのアンインストール、新しい Acrobat Pro のインストール、FRL パッケージのデプロイ、インストールの検証など、いくつかの重要な手順が含まれます。各手順を順に見ていきましょう。 |
| 「パッケージ」タブがハイライト表示された Adobe Admin Console インターフェイスのスクリーンショット | 手順 1：Admin Console の準備 | まず、システム管理者権限で Adobe Admin Console にログインします。「パッケージ」タブに移動し、「機能制限ライセンス」を選択します。「開始」ボタンをクリックして、パッケージの作成を開始します。 |
| 設定がハイライト表示されたパッケージを作成設定画面 | パッケージ設定 | 次の推奨設定でパッケージを設定します。アクティベーション方法として「オフライン」、使用権限として「PDF 生成」、プラットフォームとして「Windows 64 ビット版」を選択します。アプリの選択の場合は、選択したアプリケーションのライセンスファイルのみを保持します。 パッケージに「Acrobat FRL AEM Forms」のようなわかりやすい名前を付けて、パッケージを作成します。 |
| 権限ダイアログを含む「Admin Console ユーザー」タブ | 手順 2：ダウンロード権限を付与 | 次に、パッケージをダウンロードするユーザーを作成または特定する必要があります。Admin Console で、「ユーザー」タブに移動し、ユーザーを選択して、「デプロイメント管理者」の役割を割り当てます。これにより、作成した FRL パッケージをダウンロードできます。 |
| Acrobat を表示するアプリと機能を備えた Windows コントロールパネル | 手順 3：以前の Acrobat をアンインストール | 新しいバージョンをインストールする前に、既存の Acrobat インストールを完全に削除する必要があります。Windows コントロールパネルを開き、アプリに移動して Adobe Acrobat を見つけ、「アンインストール」を選択します。完全に削除するには、特にプロセスの後半で問題が発生した場合、Adobe Acrobat クリーナーツールの使用を検討してください。 |
| インストーラーセクションがハイライト表示された Adobe Acrobat DC ダウンロードページ | 手順 4：Adobe Acrobat Pro をダウンロードおよびインストール | アンインストール後、Adobe Acrobat DC ダウンロードページから適切な Acrobat Pro インストーラーをダウンロードします。PDF Generator の互換性を確保するには、Windows 32 ビット版インストーラーをお勧めします。 |
| 抽出と設定の手順を示すインストールプロセス | インストールプロセス | ダウンロードした zip ファイルをフォルダーに抽出し、Setup.exe を見つけて、管理者として実行します。画面の指示に従って、インストールを完了します。インストール後、Acrobat Pro を 1 回開き、ようこそダイアログを閉じて初期設定を完了します。 |
| 「ダウンロード」ボタンがハイライト表示された「Admin Console パッケージ」タブ | 手順 5：FRL パッケージをダウンロード | 次に、以前にダウンロード権限を付与したアカウントを使用して Adobe Admin Console にログインします。「パッケージ」タブに移動し、FRL パッケージを見つけて、AEM Forms サーバーにダウンロードします。 |
| コマンドを入力する際の管理者としてのコマンドプロンプト | 手順 6：パッケージをデプロイ | ダウンロードしたパッケージをサーバー上のディレクトリに抽出します。管理者としてコマンドプロンプトを開き、抽出ディレクトリに移動します。 |
| 構文がハイライト表示されたアクティベーションコマンドを示すコマンドライン | アクティベーションコマンド | `adobe-licensing-toolkit.exe -p -i -f ngl-preconditioning-data.json` の構文を使用して、アクティベーションコマンドを実行します。JSON ファイル名をパッケージの正確なファイル名に置き換えます。コマンドパラメーターは、プラットフォームの場合は -p、ライセンスのインストールの場合は -i、JSON ファイルパスの指定の場合は -f です。成功すると、「操作が正常に完了しました」というメッセージが表示されます。 |
| コマンドパラメーターと説明を示す表 | コマンドパラメーター | -p パラメーターは、プラットフォームを指定し、オペレーティングシステムを自動的に検出します。-i パラメーターは、ツールにライセンスをインストールしてアクティブ化するように指示します。-f パラメーターは、JSON ライセンスファイルへのパスを指定します。 |
| PDF Generator サービスを表示する AEM Forms 管理インターフェイス | 手順 7：テストと検証 | 最後に、AEM Forms 管理インターフェイスを開き、PDF Generator サービスを使用してシンプルなドキュメントを PDF に変換して、インストールをテストします。また、Acrobat Pro を開き、「ヘルプ」→「Acrobat Pro について」に移動し、バージョン番号とライセンスステータスを確認して、Acrobat のインストールを確認します。 |
| 分割画面：Acrobat についてダイアログと正常な PDF の変換 | 検証 | ライセンスが Acrobat でアクティブ化されていることと、PDF Generator がドキュメントを正常に変換できることを確認します。 |
| アイコン付きの一般的なエラーと解決策のリスト | トラブルシューティングのヒント | 問題が発生した場合は、次の一般的な問題を確認します。コマンドプロンプトを管理者として実行していることを確認し、JSON ファイル名が正しいことを検証し、競合するインストールを確認して、アクティベーションのためにサーバーに適切なインターネット接続があることを確認します。 |
| チェックマークが付いた追加のトラブルシューティング項目 | その他のトラブルシューティング | また、サーバー上のファイアウォールの制限、破損したパッケージ、競合する Adobe ID も確認します。それでも問題が解決しない場合は、詳細なエラーメッセージを添えてアドビサポートにお問い合わせください。 |
| チェックマークアイコンと連絡先情報が表示された成功画面 | まとめ | これで、AEM Forms サーバー上で Adobe Acrobat の機能制限ライセンスが正常に設定されました。追加のサポートが必要な場合は、アドビサポートにお問い合わせいただくか、Experience League で提供されている詳細なドキュメントを参照してください。 |
| Adobe ロゴとクレジットを含む閉じる画面 | 敬具 | このチュートリアルをご覧いただきありがとうございます。詳しくは、Experience League ドキュメントを参照してください。 |
