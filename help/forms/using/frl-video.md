# ビデオスクリプト：AEM Forms上のAdobe Acrobatの機能制限付きライセンス（FRL）の設定

| ビジュアル | タイトル | スピーカースクリプト |
|--------|-------|---------------|
| AEM Forms ロゴと FRL グラフィックを含むタイトル画面 | はじめに | AEM Forms サーバー上のAdobe Acrobatの機能制限ライセンス（FRL）の設定については、このチュートリアルをご利用ください。 Adobeが無期限のライセンスから新しい FRL モデルに移行する際には、このプロセスが不可欠です。 |
| 矢印が FRL のトランジションを指す、サポート終了（EOL）の日付を示すタイムライン | 背景 | 無期限のライセンスを持つAdobe Acrobat Classic バージョンは提供終了に達しつつあり、新しいバージョンへの移行が必要になります。 AEM Forms PDF Generatorは、ドキュメントコンバージョン機能としてAdobe Acrobatを利用します。 機能制限付きライセンスモデルは、同じ機能を提供しますが、Adobe Admin Consoleを通じて管理される最新のライセンスアプローチを備えています。 |
| チェックボックスとアイコン付きの必須項目のチェックリスト | 前提条件 | 始める前に、必要なものがすべて揃っていることを確認しましょう。まず、システム管理者権限でAdobe Admin Consoleにアクセスします。 次に、Admin Consoleのデプロイメント管理者のロールを持つユーザーアカウントです。 3 番目に、AEM Formsを実行するサーバーのローカル管理者権限です。 4 番目に、AEM Forms サーバー上の Windows 64 ビットオペレーティングシステム。 第五に、ライセンス認証のための安定したインターネット接続。 また、Adobe Admin Consoleのインターフェイスに関する知識があり、AEM Formsのデプロイメントアーキテクチャを理解している必要があります。 |
| 警告アイコンとバックアップ グラフィック | 重要な注意事項 | このプロセス中に既存のAcrobat インストールをアンインストールする必要があるので、開始する前にカスタム Acrobat設定を必ずバックアップしてください。 |
| 1 ～ 7 の番号付きステップを示すフロー図 | プロセスの概要 | このプロセスには、Admin Consoleでの FRL パッケージの準備、ダウンロード権限の付与、以前のAcrobat バージョンのアンインストール、新しいAcrobat Proのインストール、FRL パッケージのデプロイ、インストールの検証など、いくつかの重要な手順が含まれます。 各手順を説明します。 |
| 「パッケージ」タブが強調表示されたAdobe Admin Console インターフェイスのスクリーンショット | 手順 1:Admin Consoleの準備 | まず、システム管理者権限でAdobe Admin Consoleにログインします。 「パッケージ」タブに移動し、「機能制限ライセンス」を選択します。 「開始」ボタンをクリックして、パッケージの作成を開始します。 |
| 設定がハイライト表示されたパッケージ設定画面を作成 | パッケージ設定 | アクティベーション方法として「オフライン」、使用権限として「PDF Generation」、プラットフォームとして「Windows 64 ビット」を選択します。 アプリを選択する場合は、選択したアプリケーションでライセンスファイルのみを保持します。 パッケージに「Acrobat FRL AEM Forms」というわかりやすい名前を付け、パッケージを作成します。 |
| 権限ダイアログを含む「Admin Console ユーザー」タブ | 手順 2：ダウンロード権限の付与 | 次に、パッケージをダウンロードするユーザーを作成または特定する必要があります。 Admin Consoleで、「ユーザー」タブに移動して、ユーザーを選択し、「Deployment Administrator」ロールを割り当てます。 これにより、作成した FRL パッケージをダウンロードできます。 |
| Acrobatを表示するアプリと機能を備えた Windows Campaign コントロールパネル | 手順 3：以前のAcrobatをアンインストールする | 新しいバージョンをインストールする前に、既存のAcrobatのインストールをすべて削除する必要があります。 Windows Campaign コントロールパネルを開き、「アプリ」に移動してAdobe Acrobatを探し、「アンインストール」を選択します。 完全に削除するには、特にプロセスの後半で問題が発生した場合は、Adobe Acrobat クリーナーツールの使用を検討してください。 |
| インストーラーセクションがハイライト表示されたAdobe Acrobat DC ダウンロードページ | 手順 4:Adobe Acrobat Proをダウンロードしてインストールする | アンインストール後、Acrobat Pro DC のダウンロードページから適切なAdobe Acrobat インストーラーをダウンロードします。 PDF Generatorとの互換性を保つには、Windows 32 ビットインストーラーを使用することをお勧めします。 |
| 抽出とセットアップの手順を示すインストールプロセス | インストールプロセス | ダウンロードした zip ファイルをフォルダーに抽出し、Setup.exe を見つけて、管理者として実行します。 画面に表示される指示に従って、インストールを完了します。 インストール後、Acrobat Proを 1 回開いてようこそダイアログを閉じて、初期設定を完了します。 |
| 「ダウンロード」ボタンがハイライト表示された「Admin Consoleパッケージ」タブ | 手順 5:FRL パッケージのダウンロード | 次に、以前にダウンロード権限を付与したアカウントを使用してAdobe Admin Consoleにログインします。 「パッケージ」タブに移動して FRL パッケージを見つけ、AEM Forms サーバーにダウンロードします。 |
| コマンドを入力する際の管理者としてのコマンドプロンプト | 手順 6：パッケージのデプロイ | ダウンロードしたパッケージをサーバーのディレクトリに抽出します。 管理者としてコマンドプロンプトを開き、抽出ディレクトリに移動します。 |
| 構文がハイライト表示されたアクティブ化コマンドを示すコマンドライン | 有効化コマンド | 次の構文を使用してアクティブ化コマンドを実行します：`adobe-licensing-toolkit.exe -p -i -f ngl-preconditioning-data.json`。 JSON ファイル名をパッケージの正確なファイル名に置き換えます。 コマンドパラメーターは、-p （プラットフォームの場合）、-i （ライセンスのインストール時）、-f （JSON ファイルパスの指定）です。 成功すると、「操作が正常に完了しました」というメッセージが表示されます。 |
| コマンド・パラメータと説明を示す表 | コマンドパラメーター | -p パラメーターは、プラットフォームを指定し、オペレーティング システムを自動的に検出します。 -i パラメータは、ライセンスをインストールしてアクティブ化するようにツールに指示します。 -f パラメータは、JSON ライセンス ファイルへのパスを指定します。 |
| PDF Generator サービスを表示するAEM Forms管理インターフェイス | 手順 7：テストと検証 | 最後に、AEM Formsの管理インターフェイスを開き、PDF Generator サービスを使用して簡単なドキュメントをPDFに変換して、インストールをテストします。 また、Acrobatを開き、ヘルプ→バージョン情報に移動して、バージョン番号とライセンスステータスを確認して、Acrobat Proのインストールを確認します。 |
| 分割画面：Acrobatについてダイアログと正常なPDFの変換 | 検証 | ライセンスがAcrobatでアクティベート済みとして表示され、PDF Generatorがドキュメントを正常に変換できることを確認します。 |
| アイコン付きの一般的なエラーとソリューションリスト | トラブルシューティングのヒント | 問題が発生した場合は、次の一般的な問題を確認してください：管理者としてコマンドプロンプトを実行していることを確認し、JSON ファイル名が正しいことを確認し、インストールの競合を確認し、アクティブ化に適したインターネット接続がサーバーにあることを確認します。 |
| チェックマークが付いた追加のトラブルシューティング項目 | その他のトラブルシューティング | また、ファイアウォールの制限、パッケージの破損、サーバー上のAdobe ID の競合がないかも確認します。 他のすべてが失敗した場合は、詳細なエラーメッセージをAdobe サポートにお問い合わせください。 |
| 成功画面（チェックマークアイコンと連絡先情報が表示されます） | まとめ | お使いのAEM Forms サーバーで、Adobe Acrobatの機能制限ライセンスが正常に設定されました。 追加のサポートが必要な場合は、Adobe サポートに問い合わせるか、Experience Leagueで入手可能な詳細なドキュメントを参照してください。 |
| Adobeのロゴとクレジットで画面を閉じる | 敬具 | このチュートリアルをご覧いただきありがとうございます。 詳しくは、Experience League ドキュメントを参照してください。 |
